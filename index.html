JavaScript Notes: Basic to Advanced

1. Variables: var, let, and const

var: Function scoped, can be redeclared and updated. Hoisted and initialized with undefined.

let: Block scoped, can be updated but not redeclared in the same scope. Hoisted but not initialized.

const: Block scoped, cannot be updated or redeclared. Must be initialized at declaration.

// var example
var greeting = 'Hello';
var greeting = 'Hi'; // allowed: redeclaration
greeting = 'Hey'; // can update

// let example
let name = 'Alice';
// let name = 'Bob'; // Error: redeclaration not allowed
name = 'Bob'; // allowed: update

// const example
const PI = 3.14159;
// PI = 3; // Error: assignment not allowed
// const PI = 3; // Error: redeclaration not allowed

2. Data Types

Primitive Types

  • String: Sequence of characters: "Hello", 'World'
  • Number: Floating point and integer numbers: 42, 3.14
  • Boolean: true or false
  • Null: Intentional absence of any value
  • Undefined: Variable declared but not assigned
  • Symbol: Unique identifiers (ES6+)
  • BigInt: For arbitrarily large integers (ES2020+)

Reference Types

  • Object: Collections of key-value pairs
  • Array: Ordered list of values (special kind of object)
  • Function: Callable objects
  • Date, RegExp, Map, Set, etc.
typeof 'hello'    // "string"
typeof 42         // "number"
typeof true       // "boolean"
typeof undefined  // "undefined"
typeof null       // "object" (special case, historical)
typeof Symbol()   // "symbol"
typeof 10n        // "bigint"
typeof {}         // "object"
typeof []         // "object" (arrays are objects)
typeof function() {} // "function"

3. Variable Declaration and Hoisting

var declarations are hoisted and initialized with undefined, meaning you can use the variable before declaration without a runtime error (value will be undefined).

let and const declarations are hoisted but not initialized. Accessing them before declaration results in a ReferenceError - this time window is called the temporal dead zone.

console.log(x); // undefined (var hoisted)
var x = 5;

console.log(y); // ReferenceError (let / const)
let y = 10;

console.log(z); // ReferenceError (let / const)
const z = 15;

6. Scope & Closures

Scope: Local and global contexts controlling variable visibility.

Closure: When a function "remembers" variables from its defining scope even when executed elsewhere.

function outer() {
  let count = 0;
  return function inner() {
    count++;
    return count;
  }
}
const counter = outer();
console.log(counter()); // 1
console.log(counter()); // 2
